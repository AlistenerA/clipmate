# V0.5_PLAN.md — ClipMate v0.5 Article Image Saving

> v0.5 主目标：文章图片保存（Article Image Saving）

---

## 一、产品定位

让 ClipMate 在 Fullpage article clipping 时更好保留正文中的关键图片：

1. Markdown 输出中保留合规、可读、去噪后的图片语法 `![alt](imageUrl)`
2. Notion 保存时将文章图片转换为 Notion image block，优先使用 external URL
3. History 中记录图片数量和首图 URL，不保存图片二进制

---

## 二、v0.5.0 图片保存策略

```
v0.5.0 采用 external image URL 策略：
- 从文章 DOM 提取合规 img 元素的 URL
- Markdown: ![alt](url)
- Notion: image block external url
- 不下载图片二进制
- 不上传到 Notion File Upload API
- 不新增 manifest 权限
```

## 四、分 Session 计划

### Session 0：版本交接、目录创建与文章图片保存规划
- 从 clipmate-v0.4 复制创建 clipmate-v0.5
- 更新交接文档
- 明确 v0.5 图片保存边界
- 不写功能代码

### Session 1：Article Image Extraction Core
- 从文章正文 DOM 中提取 image candidates
- 支持 src/currentSrc/srcset 归一化
- 记录 alt/title/caption/width/height/sourceUrl/index
- 过滤 icon/avatar/logo/tracking/pixel/emoji/sprite/base64/blob 等噪声
- 限制数量（默认最多 20 张）
- 不下载图片
- 不新增权限

### Session 2：Markdown Image Preservation
- 将 article image candidates 安全插入或保留到 Markdown
- 去重、排序、避免重复封面图
- 保持原正文 Markdown 可读
- 增加测试

### Session 3：Notion External Image Blocks
- 将合规图片 URL 转换为 Notion image block external
- 与现有 rich_text chunk / paragraph / heading 逻辑兼容
- 失败时降级为普通链接或跳过图片，不影响正文保存
- 不实现 Notion File Upload

### Session 4：Popup / History Lightweight Image Metadata
- 显示图片数量、可选首图预览或状态提示
- History 记录 imageCount / firstImageUrl / skippedImageCount
- 不保存图片二进制，不保存 base64

### Session 5：Manual QA and Site Cases
- 测试新闻站、博客站、文档站、CSDN、知乎/微博等典型页面
- 检查图片噪声、缺图、重复图、Notion 显示
- 不扩大站点硬编码

### Session 6：Release Readiness
- 更新隐私政策、权限说明、README、测试日志
- 版本号升级到 0.5.0
- build/zip
- 发布审查

---

## 五、风险与注意事项

- 懒加载图片（data-src / lazy loading）可能无 src 属性
- srcset 归一化可能丢失多倍图
- 跨域图片 URL 可能无法在 Notion 显示
- 失效外链图片会显示 broken image
- Notion external URL 受 Notion API 限制
- base64/blob 图片需要特殊处理
- 图片数量过多可能影响保存性能
