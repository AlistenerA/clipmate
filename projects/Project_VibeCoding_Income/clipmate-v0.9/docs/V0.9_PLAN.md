# ClipMate v0.9 页面感知模式计划

更新日期：2026-06-19

## 产品目标

v0.9 让 Popup 根据当前页面结构推荐合适的剪藏模式，并在极少数高置信度场景首次自动采用推荐模式。推荐是可解释、可覆盖的导航，不是锁死用户选择的黑盒分类器。

## 优化后的范围

### P0：页面感知上下文

- 复用现有 Page Type Detector，覆盖文章、视频、讨论、AI 对话、搜索、导航和未知页面。
- 只跨消息链路传递页面类型、置信度、选区布尔值、推荐结果和固定理由；不保存原始 DOM、正文、选择文本或 detector 详细信号。
- 页面感知字段保持可选，兼容 v0.8 草稿和历史记录，无 storage schema 迁移。

### P0：推荐与用户控制

- Popup 优先显示 1-2 个相关模式，标记推荐项并解释理由。
- “更多模式”始终可以展开全文、选区、教程三种既有模式，不删除任何能力。
- 视频页把教程模式显示为“视频收藏”；讨论页把选区显示为“评论选区”；搜索/导航页把全文显示为“页面摘要”。
- 高置信度视频页和 AI 对话页首次自动采用结构化模式；恢复草稿、已有选区或用户手动切换后禁止自动覆盖。

### P0：兼容与降级

- detector、推荐或二次提取失败时继续使用全文模式。
- 不改变 content script 匹配范围、消息生命周期、Notion block 协议和 History schema。
- 不新增权限、host permission、远程 API、遥测、cookie、字幕或 DRM 访问。

## 版本节奏

- `v0.9.0-alpha.1`：纯函数推荐策略与脱敏上下文。
- `v0.9.0-alpha.2`：Popup 推荐、更多模式和可访问性。
- `v0.9.0-beta.1`：首次自动采用、草稿恢复与用户覆盖回归。
- `v0.9.0`：全量测试、构建、归档、安全检查和浏览器 QA 后冻结。

## 延后到 v0.9.x

- 独立“视频收藏”数据模型和平台 adapter。
- 用户可配置的站点级默认模式。
- 推荐反馈、遥测或云端模型。
- 字幕、登录态评论、私有 API、视频下载和 Notion File Upload。

## 验收矩阵

- 文章无代码：全文推荐。
- 技术文章含代码：教程推荐但不自动切换。
- 视频页：视频收藏推荐，高置信度首次自动采用。
- 讨论页有选区：评论选区推荐；无选区时保持可执行的全文并提示先选择评论。
- AI 对话：结构化教程推荐，高置信度首次自动采用。
- 搜索/导航：页面摘要推荐，其他模式收进“更多模式”。
- 未知/低置信度：全文降级。
- 恢复 v0.8 草稿和用户手动切换：不被自动推荐覆盖。
