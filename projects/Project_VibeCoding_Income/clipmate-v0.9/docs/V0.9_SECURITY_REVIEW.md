# ClipMate v0.9 页面感知安全检查

更新日期：2026-06-19

## 范围

本检查只覆盖 v0.9 新增的页面分类、推荐字段、Popup 模式显隐和首次自动采用。Notion Token、保存请求、Asset Picker 与 History 沿用 v0.8.5 冻结实现。

## 资产与边界

- 页面 DOM 与用户选区：不可信输入，从 content script 进入本地 detector。
- `PageAwareness`：content script 到 Popup 的扩展消息数据，也可能随草稿进入 `chrome.storage.local`。
- 模式切换：只触发既有全文、选区或教程提取，不授予权限、不新增网络目标。
- Notion Token 与用户正文：不进入推荐策略或推荐 UI。

## 主要风险与控制

| 风险 | 影响 | 控制 |
|---|---|---|
| 恶意页面 DOM 诱导误分类 | 推荐不准确、触发一次额外本地提取 | 只对高置信度视频/AI 自动采用一次；用户可覆盖；失败降级全文 |
| 页面信号扩大持久化数据 | 泄露页面结构或用户内容 | 新字段不包含 signals、DOM、正文、选区文本、URL 或 detector reasons；有自动化断言 |
| 被篡改草稿注入非法模式 | 非预期消息或状态 | `isClipMode` allowlist 拒绝非法推荐；恢复草稿不触发自动采用 |
| 被篡改推荐文案进入 UI | DOM XSS | React 文本插值自动转义并限制显示长度；无 `dangerouslySetInnerHTML` |
| 推荐锁死功能 | 用户无法使用原模式 | “更多模式”永久提供全文、选区、教程；用户选择优先级最高 |

## 权限与供应链

- v0.9 manifest 除版本号外与 v0.8.5 一致。
- package 生产依赖集合与 v0.8.5 一致。
- `npm audit --omit=dev`：0 vulnerabilities。
- 完整 audit 的 8 项告警均位于 Vite/Vitest/jsdom 开发工具链；发布包不包含这些依赖。开发服务器只绑定 `127.0.0.1`，不运行 Vitest UI。

## 结论

v0.9 页面感知没有新增高风险信任边界、权限或外部数据流。剩余主要风险是分类准确度和真实站点 DOM 差异，属于产品质量风险，应通过 Chrome/Edge 真实站点矩阵持续校准。
