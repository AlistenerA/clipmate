# NAVIGATION_SUMMARY_QA.md — ClipMate v0.4 导航摘要模式人工测试

> 本文件提供 Navigation Summary Mode 的手动测试步骤和预期结果。
> 测试对象：v0.4 Session 3.1/3.2 完成的 Navigation Summary Draft Builder + Markdown Fallback。

---

## 1. 测试目标

验证执行"剪藏当前页面"（fullpage）时：
1. 搜索结果页 / 导航页 / 低正文高链接页 能正确触发 Navigation Summary Markdown fallback
2. 普通文章页不触发
3. 视频页 / 论坛评论页 / AI 对话页不触发
4. 有用户选区的页面绝不触发（selection-first）
5. Markdown 输出安全、不含完整 DOM/正文/评论/敏感信息

---

## 2. 加载扩展

1. 打开 Edge 或 Chrome，进入 `edge://extensions/` 或 `chrome://extensions/`
2. 开启"开发人员模式"
3. 点击"加载解压缩的扩展"，选择 `clipmate-v0.4/dist/` 目录
4. 确认扩展图标出现在工具栏

## 4. 手动测试场景

### 场景 A：搜索引擎结果页

**测试页面：** Google / Bing / Baidu 任意搜索结果页

**操作步骤：**
1. 打开搜索引擎，搜索任意关键词
2. 等待结果页加载完成
3. 点击 ClipMate 图标，选择"剪藏当前页面"

**预期结果：**
- Popup 显示剪藏内容预览
- Markdown 内容以 `# 标题` 开头
- 包含 `> 当前页面更像搜索结果页，已生成主要结果摘要。`
- 包含 `- Source:` 元数据行
- 包含 `## 主要链接` 区域的编号链接列表
- 每个链接显示 `域名 — domain` 后缀
- 链接数量 ≤ 15 条

**不应出现：**
- 搜索结果页的完整 HTML
- 大量无关导航链接
- 广告链接
- 搜索输入框内容
- 搜索建议/联想列表

---

### 场景 B：普通导航/目录页

**测试页面：** 任意网站首页、导航页、目录页（如 GitHub 项目列表、新闻网站首页）

**操作步骤：**
1. 打开目标页面
2. 点击 ClipMate 图标，选择"剪藏当前页面"

**预期结果：**
- Markdown 包含 `> 当前页面更像导航/目录页，已生成主要链接摘要。`
- 包含主要链接的编号列表
- 每域名最多 3 条链接
- 总链接数 ≤ 15 条

**不应出现：**
- 导航菜单的全部项目
- 页脚链接
- 侧边栏归档链接
- 完整页面 HTML

---

### 场景 C：低正文高链接页面

**测试页面：** 内容较短但有大量内链或外链的页面（如链接聚合页、书签页）

**操作步骤：**
1. 打开目标页面
2. 点击 ClipMate 图标，选择"剪藏当前页面"

**预期结果：**
- Markdown 包含 `> 当前页面正文置信度较低，已生成安全摘要。`
- 或根据 pageType 判断，退回到旧版 fallback 消息
- 如果是 search-results/navigation pageType 且低置信，仍触发导航摘要

**不应出现：**
- 大量无意义链接
- `javascript:` / `data:` / `mailto:` 等危险协议链接
- 重复链接

---

### 场景 D：普通文章页

**测试页面：** 任意正常文章页（如技术博客、新闻正文）

**操作步骤：**
1. 打开文章页
2. 点击 ClipMate 图标，选择"剪藏当前页面"

**预期结果：**
- 正常提取文章正文
- Popup 显示正常词数统计
- **不显示** `## 主要链接` 区域
- **不显示** `- Source:` 元数据行

**不应出现：**
- 导航摘要格式的任何特征

---

### 场景 E：视频页

**测试页面：** YouTube / Bilibili / 任意视频页

**操作步骤：**
1. 打开视频页
2. 不选中任何文本
3. 点击 ClipMate 图标，选择"剪藏当前页面"

**预期结果：**
- 若为低置信触发，显示旧版 fallback 中的视频页提示：
  `> 当前页面可能是视频页，已避免保存大量媒体元素。`
- **不显示** `## 主要链接` 区域
- **不显示** `- Source:` 元数据行

**不应出现：**
- 导航摘要格式的任何特征
- 视频播放器嵌入代码
- 评论区大量楼层

---

### 场景 F：有用户选区的页面（selection-first 验证）

**测试页面：** 任意搜索结果页或导航页（预期应触发导航摘要的页面）

**操作步骤：**
1. 打开目标页面
2. **先选中一段文本**（任意内容，长度 ≥ 10 字）
3. 保持选区,点击 ClipMate 图标，选择"剪藏选区"

**预期结果：**
- 提取的是选区内容，而非导航摘要
- Markdown 只包含选中部分的文本和格式

**不应出现：**
- 导航摘要格式
- `## 主要链接` 区域

**变体测试：**
- 有选区但仍点击"剪藏当前页面"（全页模式）：应正常工作，但如果有选区且 IntentSnapshot 检测到选区，导航摘要应不触发

---

### 场景 G：论坛/评论区页

**测试页面：** Reddit 帖子、知乎回答页、微博评论区

**操作步骤：**
1. 打开评论页
2. 不选中任何内容
3. 点击 ClipMate 图标，选择"剪藏当前页面"

**预期结果：**
- 若为低置信触发，显示旧版 "评论或论坛页" 提示
- **不显示** `## 主要链接` 区域

**不应出现：**
- 导航摘要格式
- 完整楼层内容

## 5. 隐私检查

测试完成后，检查剪藏结果是否包含：

| 项目 | 应不包含 |
|------|----------|
| 完整正文全文 | ❌ |
| 完整评论全文 | ❌ |
| 完整 DOM / HTML 片段 | ❌ |
| Notion Token / API Key | ❌ |
| Notion Page ID | ❌ |
| settings 完整对象 | ❌ |
| message request 完整对象 | ❌ |
| `javascript:` / `data:` / `mailto:` 链接 | ❌ |
| 敏感个人信息 | ❌ |

---

## 6. 已知限制

1. **selector hints 为 seed 级别**：`searchResultCard` / `contentContainer` 等 selector 基于通用猜测，不承诺真实站点完整适配。某些搜索结果页可能因 DOM 结构差异而未能提取所有链接。
2. **不抓取目标链接内容**：Navigation Summary 只提取当前页面的超链接 text + href + domain，不访问链接指向的页面内容。
3. **首次 DOM 快照**：动态加载内容（如 infinite scroll）不在首次剪藏时包含。
4. **IS01 已修复**：`content/index.ts` 现在将 `assessArticleConfidence` 报告的 `articleConfidence`（数值）和 `linkDensity` 传递给 `buildLowConfidenceSummary`，低置信 + 高链接密度路径可正常触发。
5. **Notion block 转换未实现**：Navigation Summary 当前仅生成 Markdown 输出，未转换为 Notion 专用 blocks 格式。
6. **未改 Popup UI**：当前 Popup 预览区域直接显示 Markdown 原文，未做导航摘要的专属 UI。
