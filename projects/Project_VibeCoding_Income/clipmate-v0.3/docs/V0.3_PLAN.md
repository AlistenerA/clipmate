## 一、v0.3 版本定位：内容保真增强版

ClipMate v0.3 目标：把"能剪藏"升级为"剪得准、转得干净、保留结构"。

重点优化网页内容到 **Notion / Markdown / Obsidian / Typora** 的转换质量，不新增后端、AI、登录、OCR、付费。

**核心问题**：

1. LaTeX / 数学公式在 Markdown 和 Notion 中容易丢失或渲染异常。
2. 代码块常混入网页 UI，例如复制按钮、行号、折叠控件、外层提示。
3. 图片、表格、链接、摘要在不同目标中呈现不一致。
4. 导航页、搜索页、新闻页、无正文页容易错误提取。
5. Notion / Obsidian / Typora / Generic Markdown 的格式需求不同，不应使用同一套输出。

**v0.3 原则**：

- 不新增后端、登录、服务器、数据库。
- 不调用外部 AI API，不传输用户内容到第三方 LLM。
- 不新增 manifest 权限（保持 `storage` / `activeTab` / `host_permissions api.notion.com`）。
- 所有增强在现有 v0.2 架构上渐进实现。
- AI、多平台、OCR、付费延后到 v0.5+ 或后续版本。

## 三、v0.3 新 Session 拆分

| Session | 名称 | 目标 |
|:---:|------|------|
| S0 | 版本目录隔离与启动前评估 | ✅ 已完成 |

| S6 | Article Boundary Guard | 避免抓入导航、页脚、推荐、评论、无关链接；对无正文页面生成低风险摘要或提示；不做复杂站点规则库 |

## 四、v0.3 明确暂缓项（不进入本版本主线）

| 编号 | 暂缓功能 | 原因 | 建议版本 |
|:---:|------|------|:---:|
| 1 | AI 摘要 / AI 标签 / 智能备注 | 涉及模型 API、费用、隐私政策、用户内容上传、Key 管理 | v0.6+ |
| 2 | 登录系统 / 服务器 / 用户账号 | 涉及后端部署、数据库、安全、账号体系、隐私合规 | v0.6+ |
| 3 | OCR / 截图回退 | 权限、性能、准确率、包体积、审核风险较高 | v0.7+ |
| 4 | 飞书 / 语雀 / 多平台保存 | 每个平台鉴权、API、格式适配成本较高 | v0.5+ |
| 5 | 付费 / License / 订阅 | 需要后端、支付、退款、客服、合规说明 | v0.9+ |
| 6 | 图片上传、视频下载、媒体转存 | 跨域、版权、流量、存储、平台限制复杂 | v0.4-v0.5 |

## 七、每个 Session 的交付标准（继承 WORKFLOW_RULES.md）

1. `npm run lint` — 0 errors, 0 warnings
2. `npm run test` — 全部通过
3. `npm run build` — 构建成功
4. 更新交接文档（CURRENT_STATUS / CHANGELOG_AGENT / TEST_LOG / ISSUES / DECISIONS）
5. 不提交 dist/、zip、node_modules/、.env、工具目录
6. 不输出真实 Token、正文全文、备注全文、API Key
7. 发给 ChatGPT 审查后再决定是否 commit
