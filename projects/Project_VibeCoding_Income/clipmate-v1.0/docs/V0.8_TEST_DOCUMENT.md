## 3. 测试准备

1. 在 `clipmate-v0.8` 运行 `npm run build`。
2. 打开 `chrome://extensions` 或 `edge://extensions`，开启开发者模式。
3. 移除或停用旧 ClipMate，加载 `clipmate-v0.8/dist`，确认版本为 0.8.2。
4. 运行 `npm run qa:asset-picker`，打开 `http://127.0.0.1:4175/asset-picker.html`。
5. 上限、空状态和失效外链分别使用：
   - `http://127.0.0.1:4175/asset-picker-limit.html`
   - `http://127.0.0.1:4175/asset-picker-empty.html`
   - `http://127.0.0.1:4175/asset-picker-broken.html`
6. 另准备 BBC 文章、Runoob 教程、CCTV-like 图文页、Bilibili 视频页和一个测试 Notion 页面。
7. 测试记录不得包含 Notion token、Page ID、用户正文或完整 Markdown。

### 6.1 Runoob v0.8.2 专项复测

1. 打开 TypeScript 教程并切换“教程”模式。
2. 在 Popup 预览中找到“实例”代码：代码应在独立代码框内，换行保留；不得与“尝试一下”链接混成普通段落。
3. 找到“1. 声明部分”：预览和 Notion 都应显示 `1.`，不得显示 `1\.`。
4. 找到“TypeScript 的主要优势”：Popup 保持项目符号；保存到 Notion 后每项应为原生列表，粗体前缀正常，不得整行斜体或残留单独 `*`。
5. 找到没有 alt/题注的正文图：图片下方不得自动出现 `image` 或“网页图片”题注。

## 8. 回归与安全检查

- 全文、选区、教程提取、预览、复制、保存、History、重试均需通过。
- Options 的 token、目标 CRUD、默认目标、历史筛选与删除均需通过。
- Popup、Options、content script、service worker 控制台无新增错误。
- Manifest 只允许 `storage`、`activeTab`；host permission 只允许 Notion API。
- Asset Picker 不读取 cookie，不调用第三方私有 API，不下载/上传图片，不建立站点级长期缓存。
- 页面文字、token、Page ID、Notion 响应正文不得进入日志或测试记录。
