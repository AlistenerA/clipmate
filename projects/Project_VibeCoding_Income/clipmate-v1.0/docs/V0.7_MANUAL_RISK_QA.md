# ClipMate v0.7 教程模式人工风险测试

更新日期：2026-06-18

## 1. 测试目标

本清单验证自动化测试无法充分覆盖的浏览器扩展链路：Popup 状态、真实页面 DOM、content script、MV3 background、真实 Notion API、Chrome/Edge 差异和外部图片可访问性。

测试版本必须满足：

- 分支：`codex/clipmate-v0.7-tutorial-mode`
- 目录：`clipmate-v0.7/`
- manifest 版本：`0.7.0`
- 加载目录：`clipmate-v0.7/dist/`

## 2. 测试前准备

1. 在 `clipmate-v0.7` 运行 `npm run build`。
2. Chrome 打开 `chrome://extensions`，Edge 打开 `edge://extensions`。
3. 开启开发者模式，选择“加载已解压的扩展程序”，加载 `clipmate-v0.7/dist`。
4. 在 `clipmate-v0.7` 运行 `npm run qa:tutorial`。
5. 打开 `http://127.0.0.1:4174/tutorial-mode.html`。
6. 准备一个仅用于测试的 Notion 页面；不要在测试记录中粘贴 Token 或 Page ID。

## 3. P0 必测项

| ID | 风险 | 操作 | 预期结果 |
|---|---|---|---|
| T07-01 | 模式切换保存了旧内容 | 打开样例页，依次点击“全文”“选区”“教程” | 每次点击立即重新提取；加载期间旧内容消失且不能误保存；状态栏显示当前模式 |
| T07-02 | 教程结构在提取阶段丢失 | 选择“教程”，查看原文与预览 | 存在 H1/H2/H3、TypeScript 代码、`E = mc^2`、两列表格、TIP、图片与题注、Video links |
| T07-03 | Markdown 复制降级 | 目标依次选 Notion、Obsidian、Typora、通用并复制 | 各 profile 的头部规则正常；正文教程结构不被压平成单段；粗体后空格规则无回归 |
| T07-04 | Notion block 类型错误 | 保存样例页到测试 Notion 页面 | 标题层级对应 heading；代码为 code；公式为 equation；表格为 table；TIP 为 callout；图片题注在 caption；视频为 bookmark |
| T07-05 | 全文模式被新模型影响 | 在普通新闻或博客页用“全文”保存 | 行为与 v0.6 一致；不会强制生成教程表格、equation 或视频 bookmark |

| T07-07 | History 无法识别新 mode | 教程模式保存成功后打开 Options > History | 记录显示“教程”；标题、URL、图片数、保存状态正确；重试不报 mode 错误 |

## 4. P1 高风险补测

| ID | 风险 | 操作 | 预期结果 |

| T07-13 | 草稿恢复错位 | 教程提取后关闭并重新打开 Popup | 同 URL 恢复标题、标签、备注、教程 mode 和结构化文档；换 URL 不恢复旧草稿 |
| T07-14 | 错误态不可恢复 | 在受限页、无选区页、content script 未注入页操作 | 显示中文错误；切回正常网页可重新提取；Popup 无未处理异常 |
| T07-15 | 浏览器差异 | 在 Chrome 和 Edge 各完成 T07-01、02、04、07 | 两个浏览器结果一致；若不同，记录浏览器版本和扩展控制台错误 |

## 5. 真实站点建议

至少选三类页面：

- 技术文档：含多级标题、代码块、表格。
- 数学或科研文章：含 MathJax/KaTeX 公式。
- 视频教程文章：正文中含 YouTube、Bilibili 或 Vimeo 嵌入链接。

不要在测试文档中保存真实用户正文、评论全文、Token、Page ID 或账号信息。只记录站点域名、结构类型和失败现象。

## 7. 发布判定

- P0 任一失败：v0.7 不应发布，先修复并回归相关用例。
- P1 数据丢失、重复保存、Notion 整页失败或隐私泄露：按 P0 处理。
- 外部图片仅因源站防盗链失败：记录为已知限制，但正文必须保存成功。
- Chrome/Edge + 真实 Notion QA 未执行时，只能标记“自动化候选版”，不能标记“人工验收完成”。
