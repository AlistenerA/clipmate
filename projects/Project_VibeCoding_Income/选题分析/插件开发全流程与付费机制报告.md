## 一、浏览器插件开发到上架全流程

### 1.1 技术栈选择

| 决策项 | 推荐方案 | 说明 |
|--------|----------|------|
| 开发语言 | TypeScript | 类型安全，所有竞品均使用 TS/JS |
| UI 框架 | React 18 | web-clipper 已验证 React 在插件中的可用性 |
| 打包工具 | Vite + @crxjs/vite-plugin | 比 webpack 开发体验好，支持 HMR |
| 状态管理 | Zustand | 轻量级，比 Redux 适合插件场景 |
| CSS 方案 | Tailwind CSS | 快速开发 UI，体积小 |
| 代码规范 | ESLint + Prettier | 保持代码质量 |
| 测试框架 | Vitest | 与 Vite 生态兼容 |

### 1.2 Manifest V3 核心架构

```json
{
  "manifest_version": 3,
  "name": "__MSG_extName__",
  "description": "__MSG_extDesc__",
  "version": "0.1.0",
  "default_locale": "zh_CN",
  "icons": { "16": "...", "48": "...", "128": "..." },
  "action": {
    "default_popup": "popup.html",
    "default_icon": "..."
  },
  "background": {
    "service_worker": "background.js"
  },
  "content_scripts": [{
    "matches": ["<all_urls>"],
    "js": ["content.js"],
    "run_at": "document_idle"
  }],
  "permissions": [
    "storage",      // 本地存储用户设置和授权
    "activeTab",    // 获取当前标签页内容
    "scripting",    // 动态注入脚本
    "tabs"          // 标签页信息
  ],
  "host_permissions": ["<all_urls>"],
  "web_accessible_resources": [{
    "resources": ["assets/*"],
    "matches": ["<all_urls>"]
  }]
}
```

**关键概念：**

| 概念 | 说明 |
|------|------|
| Service Worker | 替代 MV2 的 background page，事件驱动，非持久化 |
| Content Script | 在网页上下文中运行的脚本，可访问 DOM |
| Popup | 点击工具栏图标弹出的 UI |
| Message Passing | Service Worker ↔ Content Script 通信桥梁 |

### 1.3 开发阶段（Day 1-Day 4）

```
阶段一：项目脚手架搭建 (半天)
├── 初始化 Vite + React + TypeScript 项目
├── 配置 @crxjs/vite-plugin (Manifest V3)
├── ESLint + Prettier + Husky
└── 基本目录结构搭建

阶段二：Content Script 开发 (1天)
├── 内容提取器 (基于 Mozilla Readability 库)
├── DOM 选择器 (类似简悦的鼠标框选)
├── 页面信息收集 (URL、标题、元数据)
└── 消息通信基础设施

阶段三：Popup UI 开发 (1天)
├── 目标选择器 (工作区/页面/数据库)
├── 剪藏预览面板
├── 标签/属性编辑
└── 保存确认与结果反馈

阶段四：存储与目标平台对接 (1天)
├── Notion API 集成
├── 飞书/语雀 API 集成（可选）
├── Markdown 格式导出
└── 本地缓存与离线队列

阶段五：设置页 (Option Page) (半天)
├── API Key 管理
├── 默认目标设置
├── 剪藏规则配置
└── 数据导出/导入
```

### 1.4 Edge Add-ons 上架流程（Day 5-Day 7）

基于 Microsoft Learn 官方文档（2026年5月更新），完整步骤如下：

**Step 1: 准备提交**
- ✅ 可运行的扩展原型（`.zip` 包）
- ✅ 注册 [Partner Center](https://partner.microsoft.com/dashboard/microsoftedge/) 开发者账号
- ✅ 支付一次性注册费 $19（约￥140）
- ✅ 准备隐私政策 URL（必须）

**Step 2: 创建新扩展**
- 进入 Partner Center > Edge > 创建新扩展
- 上传 `.zip` 包（含 `manifest.json` + 所有资源文件）

**Step 3: 配置 Available（可用性）**
- `Visibility`: Public（公开）或 Hidden（隐藏分享）
- `Markets`: 默认全球所有市场（可选中国等特定市场）

**Step 4: 填写 Properties（属性）**
- Category: Productivity（生产力工具）
- Website URL（可选）
- Support contact detail（支持联系）
- Mature content check（成人内容声明）

**Step 5: 填写 Privacy（隐私）** — 2026年5月新增独立页面
- **Single Purpose Description**: 清晰描述核心功能
- **Permission Justification**: 逐一说明每个权限的原因
- **Remote Code**: 声明是否使用远程代码（MV3 禁止远程执行代码）
- **Data Usage**: 收集哪些用户数据、用途说明
- **Privacy Policy URL**: 必填（若收集个人信息）

**Step 6: 填写 Store Listing（商店列表）**
- Extension name（扩展名称）
- Description（详细描述，最少250字符，最多10000字符）
- 支持 AI 自动生成描述（限10次/小时）
- Extension logo（300×300px，最少128×128px）
- Small promotional tile（440×280px）
- Large promotional tile（1400×560px）
- Screenshots（最多6张，640×480 或 1280×800）
- YouTube promotional video（可选）
- Search terms（可选，最多7个搜索词，每个最多30字）

**Step 7: 提交审核（Submit）**
- 填写 Notes for certification（认证说明，含测试账号等）
- 点击 Publish 提交
- 认证过程最长 **7 个工作日**
- 通过后状态变为 "In the Store"

**Step 8: 更新维护**
- 支持延迟发布（Deferred Publish）：审核通过后手动选择时机发布（最长保留30天）
- 支持取消审核（Cancel Review）
- 支持版本回滚（Rollback）
- 支持编程式发布（Chrome Web Store Publish API）

**完整时间线预估：**

| 阶段 | 内容 | 时间 | 累计 |
|------|------|------|------|
| Day 1 | 项目脚手架 + Content Script 基础 | 1天 | 1天 |
| Day 2 | Content Script 完善 + Popup UI | 1天 | 2天 |
| Day 3 | 目标平台对接 + 设置页 | 1天 | 3天 |
| Day 4 | 测试 + 调试 + Bug 修复 | 1天 | 4天 |
| Day 5 | 素材准备 + Partner Center 填写 | 1天 | 5天 |
| Day 6-11 | 审核期（等待，可做营销准备） | 1-7工作日 | - |
| Day 12 | 上架 + 发布营销 | 1天 | - |

> **关键风险:** 审核时间为 1-7 个工作日，首次提交建议预留一周。Edge Add-ons 审核比 Chrome Web Store 更为严格。

### 1.5 同时上架 Chrome Web Store

Edge 基于 Chromium，同一份代码可直接提交 Chrome Web Store：
- 注册费 $5（一次性）
- 审核时间 1-3 个工作日
- Manifest V3 通用
- 用户基数更大

## 二、付费机制调研

### 2.1 官方渠道付费现状

| 平台 | 付费模式 | 状态 |
|------|----------|------|
| Chrome Web Store | Chrome Web Store Payments | **已弃用（2024年起）** |
| Chrome Web Store | 付费扩展 (one-time payment) | **已停止接受新付费扩展** |
| Edge Add-ons | Partner Center 内置支付 | **不支持** |
| Edge Add-ons | 外部支付 | **允许（需声明）** |

**结论：两大官方商店均不支持内置支付，必须通过外部方案实现付费。**

### 2.2 行业主流付费模式对比

| 模式 | 实现方式 | 优点 | 缺点 | 适用场景 |
|------|----------|------|------|----------|
| **免费增值 (Freemium)** | 基础功能免费 + 高级功能付费 | 转化路径短，用户增长快 | 付费率低（通常2-5%） | 功能型插件首选 |
| **订阅制 (Subscription)** | 按月/年付费 | 持续收入，可预测 | 用户决策门槛高 | 高价值专业工具 |
| **买断制 (One-time)** | 一次性支付终身使用 | 用户决策快 | 持续收入有限 | 功能稳定的工具 |
| **License Key** | 外部网站购买后输入激活码 | 灵活定价，无平台抽成 | 需自建支付验证系统 | 当前主流方案 |
| **增值服务** | 云同步/API 调用次数付费 | 与使用量挂钩 | 用户感知成本不透明 | AI/云服务类插件 |

### 2.3 外部支付方案实现

```
用户流程：
1. 安装免费插件
2. 使用基础功能 → 触发付费墙
3. 点击升级 → 跳转外部支付页（Stripe/Paddle/Lemon Squeezy）
4. 完成支付 → 自动发送 License Key 到邮箱
5. 在插件内输入 License Key → 验证 → 解锁高级功能

技术架构：
┌─────────────┐     ┌──────────────┐     ┌─────────────┐
│  浏览器插件  │ ←→  │  后端 API 服务 │ ←→  │  支付服务商  │
│  (前端验证)  │     │  (License验证)│     │ (Stripe等)  │
└─────────────┘     └──────────────┘     └─────────────┘
```

### 2.4 竞品付费策略调研

| 插件 | 付费模式 | 价格 | 收入预估 |
|------|----------|------|----------|
| **简悦 (SimpRead)** | Freemium + 付费订阅 | ¥6/月 或 ¥60/年 | 保守估计月收入 ¥5-10万（8.6k stars × 约3%付费） |
| **Cubox** | 纯订阅 | ¥98/年 | 估算年收入百万级 |
| **Save to Notion** | 免费 | 免费 | 无直接收入，通过流量导向其他服务 |
| **Web Clipper (webclipper)** | 开源免费 | 免费 | 无收入，纯社区维护 |
| **Notion Web Clipper** | 免费 | 免费 | 作为 Notion 生态的一部分 |

### 2.5 推荐付费策略

**针对新插件的「三段式」付费方案：**

| 层级 | 功能 | 推荐价格（¥） | 目标转化率 |
|------|------|:---:|:---:|
| **Free** | 基础剪藏（HTML/纯文本）、Notion 保存、1个目标工作区 | 免费 | 100% |
| **Pro** | Markdown 导出、选择性剪藏、多工作区、标签系统、自定义模板、图片 OCR 识别 | ¥9.9/月 或 ¥69/年 | 5-8% |
| **Team** | 团队共享、API 配额提升、优先支持、高级导出（飞书/语雀等） | ¥29/月 或 ¥199/年 | 1-2% |

**实施建议：**
1. **前3个月免费期**：所有功能免费，快速积累用户和口碑
2. **宽松的免费层级**：免费用户功能足够日常使用，不至于因功能过少而卸载
3. **清晰的价值差异**：Pro 功能应能明显提升效率（如 OCR、AI 摘要）
4. **年付折扣**：年付打 5-6 折，锁定长期用户

### 2.6 支付服务商选择

| 服务商 | 手续费 | 优点 | 缺点 |
|--------|:---:|------|------|
| **Lemon Squeezy** | 5% + $0.50 | 一站式（含税务、发票），适合独立开发者 | 手续费偏高 |
| **Stripe** | 2.9% + $0.30 | 手续费低，API 强大 | 中国商户开户困难 |
| **Paddle** | 5% + $0.50 | 支持中国实体，含税务处理 | 审核严格 |
| **爱发电/Afdian** | 6% | 国内支付方便（微信/支付宝） | 仅国内用户 |
| **PayPal** | 4.4% + $0.30 | 国际化 | 国内用户使用不便 |

> **最佳组合**: Lemon Squeezy（国际用户）+ 爱发电（国内用户），两条线并行。
