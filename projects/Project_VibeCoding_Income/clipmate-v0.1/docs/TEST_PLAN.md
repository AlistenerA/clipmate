# TEST_PLAN.md — ClipMate v0.1 测试计划

> 本轮为 Session 5 最终验收，目标：确认 v0.1 所有功能可用，打包产物可正常加载。

---

## 测试环境

| 项目 | 要求 |
|------|------|
| 浏览器 | Microsoft Edge ≥ 126（Chromium 内核） |
| 扩展模式 | 开发者模式，加载解压缩的扩展 |
| 加载路径 | `clipmate-v0.1/dist/` |
| Notion Token | 有效的 Integration Token（[创建地址](https://www.notion.so/my-integrations)）|
| 目标 Page | Token 有写入权限的 Notion 页面 |
| 测试页面 | 见各用例说明 |

---

## TC-01：安装测试

**目标**：确认构建产物可被 Edge 加载。

1. 打开 `edge://extensions`，开启「开发人员模式」
2. 点击「加载解压缩的扩展」，选择 `clipmate-v0.1/dist/`
3. 预期：
   - 扩展出现在列表中，名称为「ClipMate」
   - 无红色错误提示
   - 版本号显示 `0.1.0`

**通过条件**：扩展加载成功，无错误。

---

## TC-02：Edge 加载 dist 测试

**目标**：确认扩展所有入口页面可正常打开。

1. 在已加载的扩展中，确认工具栏出现 ClipMate 图标
2. 点击图标，Popup 正常弹出显示
3. 右键点击图标 → 「扩展选项」，Options 页在标签页中打开

**通过条件**：Popup 和 Options 页面均可正常显示，无白屏、无崩溃。

---

## TC-03：Popup 打开测试

**目标**：确认 Popup UI 各元素正常渲染。

1. 在任意网页（如 `https://www.example.com`）点击 ClipMate 图标
2. 预期显示：
   - 模式切换按钮（全文 / 选区）
   - 状态栏（含字数统计）
   - 内容预览区域
   - 标签输入框
   - 备注输入框
   - 「复制 Markdown」按钮
   - 「保存到 Notion」按钮
   - 设置图标（齿轮）

**通过条件**：所有 UI 元素正常渲染，无布局错乱。

---

## TC-04：Options 保存配置测试

**目标**：确认设置页面可正确读写配置。

1. 打开 Options 页面（右键图标 → 扩展选项）
2. 填写以下配置：
   - Notion Integration Token：`secret_xxx`（有效 Token）
   - Notion Page ID：`xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`
   - 默认标签：`测试,test`
   - 启用/禁用标签历史
3. 点击「保存」，预期显示「设置已保存」
4. 关闭 Options 页面，重新打开
5. 预期：所有配置保持不变

**通过条件**：配置保存后不丢失，重新打开正确还原。

---

## TC-05：Fullpage 正文提取测试

**目标**：确认全文模式正确提取网页正文。

1. 打开一篇博客文章（如 `https://zhuanlan.zhihu.com/p/` 任意公开文章）
2. 点击 ClipMate 图标，选择「全文」模式
3. 预期：
   - 状态栏显示文章标题
   - 字数统计 > 0
   - 内容预览显示正文（不含导航栏、侧边栏、广告）
   - 页面元数据（标题、描述）正确识别

**通过条件**：正文提取去除了导航/广告等非内容元素。

---

## TC-06：Selection 选区剪藏测试

**目标**：确认选区模式正确提取用户选中文字。

1. 打开任意网页，用鼠标选中一段文字（不少于 10 个字）
2. 点击 ClipMate 图标，选择「选区」模式
3. 预期：
   - 状态栏显示选中文字的字数
   - 内容预览显示选中的文字
   - 选中文字中的 HTML 格式（粗体、链接）保留为 Markdown

**通过条件**：选区文字被正确提取，字数统计准确。

---

## TC-07：复制 Markdown 测试

**目标**：确认复制 Markdown 包含完整元数据。

1. 在全文模式下提取一篇文章
2. 添加标签（如 `技术,前端`）
3. 填写备注（如 `值得收藏`）
4. 点击「复制 Markdown」
5. 打开任意文本编辑器（如记事本），粘贴
6. 预期内容结构：
   ```
   # 文章标题
   来源：https://...
   标签：#技术 #前端
   > 值得收藏
   ---
   [正文内容]
   ```
7. 确认正文中无 `****` 异常粗体标记

**通过条件**：粘贴结果包含标题、URL、标签、备注、分割线、正文六部分，无异常粗体。

---

## TC-08：保存到 Notion 测试

**目标**：确认剪藏内容成功保存到 Notion 页面。

1. 在 Options 已配置有效 Token 和 Page ID
2. 在全文模式下提取一篇文章
3. 添加标签和备注
4. 点击「保存到 Notion」
5. 预期：
   - 按钮显示「保存中...」
   - 成功后显示「已保存到 Notion」
   - 打开目标 Notion 页面，底部追加了新内容
6. 检查 Notion 页面中：
   - 标题正确
   - URL 以链接形式显示
   - 标签以蓝色文字显示
   - 备注在 📝 callout 中
   - 正文以段落形式出现
   - 有分割线（divider）

**通过条件**：Notion 页面中内容结构完整，格式正确。

---

## TC-09：无选区错误测试

**目标**：确认无选区时给出正确提示。

1. 在任意网页，不选中任何文字
2. 点击 ClipMate 图标，切换到「选区」模式
3. 预期：
   - 显示错误提示「请先选中页面文字后再提取」
   - 不显示内容预览

**通过条件**：显示用户友好的中文错误提示。

---

## TC-10：Token 缺失测试

**目标**：确认 Token 未配置时给出正确提示。

1. 打开 Options 页面，清空 Notion Token，保存
2. 提取任意网页内容
3. 点击「保存到 Notion」
4. 预期：
   - 显示错误提示「请先在设置中配置 Notion Token」
   - 不发起 API 请求

**通过条件**：显示用户友好的中文错误提示。

---

## TC-11：Token 错误测试

**目标**：确认 Token 无效时给出正确提示。

1. 在 Options 中填写一个明显无效的 Token（如 `secret_invalid12345`）
2. 提取网页内容，点击「保存到 Notion」
3. 预期：
   - 显示错误提示「Notion 授权失败，请检查 Token 是否正确」
   - 不会将 Token 打印到控制台

**通过条件**：用户友好的错误提示，Token 不泄露到任何可见输出。

---

## TC-12：Page ID 错误测试

**目标**：确认 Page ID 无效时给出正确提示。

1. 在 Options 中填写正确的 Token，但 Page ID 填写不存在的 ID（如 `00000000-0000-0000-0000-000000000000`）
2. 提取网页内容，点击「保存到 Notion」
3. 预期：
   - 显示错误提示「Notion 页面不存在，请检查 Page ID」
   - 或「Notion 授权失败」（取决于 Notion API 返回的具体错误）

**通过条件**：用户友好的错误提示，不崩溃。

---

## TC-13：Integration 未授权页面测试

**目标**：确认 Integration 未添加到目标页面时给出正确提示。

1. 在 Notion 中创建一个新页面，不将 Integration 添加到该页面的 Connections
2. 使用该页面的 Page ID 尝试保存
3. 预期：
   - 显示错误提示（授权失败或页面不存在）

**通过条件**：用户友好的错误提示。

---

## TC-14：当前页面不允许注入 Content Script 测试

**目标**：确认在受保护页面（如 `edge://`、`chrome://`）不会崩溃。

1. 打开 `edge://extensions` 页面
2. 点击 ClipMate 图标
3. 预期：
   - Popup 正常打开
   - 显示提示信息说明无法在该页面提取内容

**通过条件**：不崩溃，有合理提示。

---

## TC-15：隐私与日志测试

**目标**：确认控制台不泄露敏感信息。

1. 打开 DevTools（F12），切换到 Console 标签
2. 执行 TC-08（保存到 Notion）和 TC-11（Token 错误）
3. 检查控制台输出：
   - 不应出现明文 Token
   - 不应出现剪藏的正文全文
   - 不应出现用户备注全文
   - 可以出现错误码（如 `NOTION_AUTH_FAILED`）
   - 可以出现字数统计

**通过条件**：控制台无 Token、正文、备注等敏感信息泄露。

---

## TC-16：Build 打包测试

**目标**：确认 `npm run build` 产出可用的 dist 目录。

1. 运行 `npm run build`
2. 检查 `dist/manifest.json` 存在且内容正确
3. 检查 `dist/` 下包含：
   - `manifest.json`
   - `service-worker-loader.js`
   - `assets/` 目录（JS/CSS 文件）
   - `src/popup/index.html`
   - `src/options/index.html`
4. 根据打包说明生成 zip 文件
5. 确认 zip 内不包含：
   - `src/` 源码文件（非 HTML）
   - `node_modules/`
   - `tests/`
   - `docs/`
   - `.git/`

**通过条件**：Build 成功，zip 产物不包含源码。
