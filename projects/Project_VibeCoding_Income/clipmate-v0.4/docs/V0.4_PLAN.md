# ClipMate v0.4 Plan — Site Profiles and Scenario Modes

## Version Positioning

ClipMate v0.4 focuses on site profiles and scenario modes.

中文定位：站点适配与场景模式版

Goal:
The extension should better detect what kind of page the user is clipping and choose a safer extraction mode based on site-specific rules and universal heuristics.

## Baseline

- Source version: clipmate-v0.3
- v0.3 final code commit: d5d771d fix(clipmate): finalize v0.3 manual qa fixes
- v0.4 handoff commit: 97acbc9 docs(clipmate): add v0.4 handoff after v0.3 public snapshot
- package.json version currently inherited from v0.3: 0.3.0
- manifest.config.ts version currently inherited from v0.3: 0.3.0
- Version bump to 0.4.0 is deferred to v0.4 release docs/package session.

## Core Problems Carried from v0.3

- CSDN / LaTeX site-specific rendered formula residue
- Search result page and navigation page classification quality
- No structured page type detector
- No site profile engine
- No dedicated navigation summary mode
- No dedicated link card mode

## v0.4 Middle Target

v0.4 的中间目标从单纯 Site Profiles 扩展为：

**Page Type + Site Profile + Intent Snapshot + Clip Strategy**

即：先识别页面类型 → 匹配站点规则 → 结合用户选区/可见区域/交互意图 → 选择剪藏策略。

---

## v0.4 Risk Areas

### 短视频 / 评论区场景

- 短视频和评论密集站点的 DOM 高度动态（TikTok/Douyin/Xiaohongshu 等）
- 需要 Intent Snapshot 辅助判断，不能仅依赖 PageType
- Session 2.2 Seed Site Profiles 必须涵盖此类站点
- Session 4 Comment / Selection Clip Mode 需要结合 Intent

### 无法判断的用户行为

- 信号不足或冲突严重的场景保留为 `unknown`
- 当前无 AI 能力但未来可能有帮助的场景标记 `needs-ai-later`
- 不强猜用户意图

## v0.4 Scope

### Session 1 — Page Type Detector ✅ 已完成

Detect page types:

- article
- search-results
- navigation
- forum-or-comment
- video
- ai-answer
- unknown

### Session 1.1 — Intent & Site Profile Planning ✅ docs-only

产出：

- `SITE_INTENT_MATRIX.md` — 站点意图矩阵（seed site profiles、intent 类型、IntentSnapshot 概念）
- `QUALITY_GUARDRAILS.md` — 质量门禁（vibe slop 防范、模块边界、Anti-Slop Review 节点）

### Session 2 — Site Profile Engine

Create a structured site profile layer:

- domain / hostname / URL pattern matching
- no scattered hard-coded site if statements
- safe fallback to generic extractor

### Session 2.1 — Intent Signal Collector（建议新增）

基于 SITE_INTENT_MATRIX.md 实现：

- 综合 PageType + SiteProfile + SelectionContext + VisibleContext + RecentInteraction
- 输出 IntentSnapshot（不持久化，仅 tab 内存）
- 无法判断时降级 unknown / needs-ai-later

### Session 2.2 — Seed Site Profiles（建议新增）

基于 SITE_INTENT_MATRIX.md 的重点站点列表：

- 创建 Google/Bing/Baidu/YouTube/Bilibili/Zhihu/Reddit/微博 等结构化 profile
- 每个 profile 为数据配置，非业务逻辑硬编码

### Session 2.3 — Anti-Slop Review（建议新增）

Profile + Intent 完成后质量审查：

- 审核散落 domain if
- 审核模块边界
- 审核测试和日志

### Session 3 — Navigation Summary Mode

For non-article pages:

- show clear non-article warning
- keep title, URL, domain
- keep limited clean links
- avoid saving massive navigation noise

### Session 4 — Comment / Selection Clip Mode

For comment-heavy pages:

- prioritize user selection
- avoid infinite comment/thread extraction
- keep source and context

### Session 5 — Site Icon / Theme Cache

Improve visual identity:

- favicon fallback
- site initial fallback
- safe local cache
- no remote analytics

### Session 6 — Link Card Preview

For URL/link-centric pages:

- title
- domain
- summary
- favicon
- source URL

### Session 7 — Docs, Manual QA, Version Bump, Package

- update version to 0.4.0
- update release docs
- update manual QA
- run lint/test/build/zip

### Session 8 — Release Robustness Review

- permission review
- privacy review
- storage review
- token/logging review
- zip content review

### Deferred — History UX / Settings Refactor

以下功能从 v0.4 当前路线移出，延后到 v0.5 或独立 History UX / Settings Refactor 版本：

- **Tag Search UX**：历史搜索栏支持 #tag，快速筛选标签。
- **Better History Config**：历史入口放到设置页或独立 tab，不继续挤在 Popup 主流程。

原因：
- 两者属于历史记录/设置页体验增强，不是 v0.4 PageType + SiteProfile + Scenario Modes 的核心闭环。
- Better History Config 可能牵涉 Popup / Options UI 重构，容易扩大 v0.4 范围。
- 延后不影响 Navigation Summary、Comment / Selection、Site Icon、Link Card、发布 QA。

## Strict Non-goals

- No login system
- No backend server
- No AI API
- No OCR
- No cloud sync
- No paid feature
- No media download
- No new manifest permissions without prior ChatGPT review
- No remote JavaScript

## Permission Policy

v0.4 should keep current permissions unless a dedicated review approves changes:

- permissions: storage, activeTab
- host_permissions: https://api.notion.com/*
- content_scripts.matches: <all_urls>

## Commit Policy

- Agent never commits by default.
- User sends output to ChatGPT for review.
- ChatGPT decides whether to provide a precise commit prompt.
- Never use `git add .`.
- Never push from the development repository unless explicitly requested.
