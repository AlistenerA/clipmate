## v0.4 Recommended Positioning

**ClipMate v0.4 should focus on Site Profiles and Scenario Modes.**

Goal: The extension should better understand what kind of page the user is clipping and choose a safer extraction mode based on site-specific rules and universal heuristics.

### Recommended v0.4 Sessions

1. **Session 0** — v0.4 planning and workspace creation review (ChatGPT review before starting)
2. **Session 1** — Page Type Detector (enhanced classifyPageType with site profiles)
3. **Session 2** — Site Profile Engine (per-site extraction rules, container selectors, noise patterns)
4. **Session 3** — Navigation Summary Mode (improved low-confidence summary for nav/search pages)
5. **Session 4** — Comment / Selection Clip Mode (select and clip specific content blocks)
6. **Session 5** — Tag Search UX (search/filter tags in history and popup)
7. **Session 6** — Site Icon / Theme Cache (cache favicons and theme colors for history display)
8. **Session 7** — Link Card Preview (rich link previews in Notion blocks)
9. **Session 8** — Docs, QA, version bump, package
10. **Session 9** — Robustness check and release candidate review

## v0.4 Strict Non-goals

- No login system
- No backend server
- No AI API (OpenAI/DeepSeek/Claude/Zhipu/Gemini)
- No OCR / screenshot fallback
- No media download (images, videos, PDFs)
- No cloud sync
- No paid features or license keys
- No new manifest permissions without prior review
- No remote JavaScript execution
