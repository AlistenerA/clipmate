# ClipMate v1.0.2 社区站点剪藏计划

更新日期：2026-06-21

## 目标

为小红书图文笔记与百度贴吧主题提供站点专用全文剪藏，优先保存用户真正看到的帖子主体、图片和评论讨论，降低动态页面在通用 Readability 下的正文缺失与链接污染。

## 输出结构

### 小红书

1. 标题、作者、发布时间、来源。
2. 正文与主题标签。
3. 轮播图按页面顺序去重保存。
4. 评论区按 DOM 顺序最多 50 条，含可见的作者、正文、时间和嵌套回复关系。

### 百度贴吧

1. 主题标题、吧名/来源。
2. 楼主和后续楼层正文、图片、楼层号/时间（页面可见时）。
3. 楼中楼回复计入最多 50 条评论/回复上限。

## 边界

- 只读取当前已渲染 DOM，不调用私有 API，不读取 Cookie，不自动登录。
- 不下载、缓存或重新上传图片；继续使用 URL-only 图片策略。
- 不保存导航、推荐、广告、关注、点赞/收藏/评论统计跳转等无关链接。
- 不新增 manifest 权限，不改变 License、Notion 或 storage schema。

## 验证

- 基于脱敏 fixture 的小红书/贴吧单元测试：正文、图片顺序、链接过滤、嵌套评论与 50 条硬上限。
- 现有全量 lint/test/build 回归。
- Playwright 使用有头窗口；用户自行登录后观察真实页面 DOM，并验证 built extension 的实际剪藏预览。
