# ClipMate v1.0.2 浏览器验收清单

更新日期：2026-06-21

## 环境

- 只加载 `clipmate-v1.0.2/dist/` production 构建。
- 浏览器窗口必须可见；登录由用户本人操作。
- 不记录账号、Cookie、完整页面 DOM 或带登录凭证的查询参数。

## 小红书

1. 打开已登录的 `/explore/*` 图文笔记并启动 ClipMate 全文提取。
2. 预览包含标题、作者、正文和按页面顺序排列的轮播图片。
3. 预览包含可见评论与嵌套回复，总数不超过 50。
4. Markdown 只有一个“原帖”链接，且 URL 不含 `xsec_token` 等查询参数。
5. 关注、点赞、收藏、评论统计、推荐和用户主页不形成输出链接。

## 百度贴吧

1. 打开已登录的 `/p/<数字>` 主题并启动 ClipMate 全文提取。
2. 预览包含主题标题、首楼正文和正文图片。
3. 预览包含楼层回复与楼中楼回复，总数不超过 50。
4. Markdown 只有一个“原帖”链接，且 URL 不含 `fr` 等查询参数。
5. 吧导航、用户主页、推荐主题和交互统计不形成输出链接。

## 交互与回归

1. 提取完成后页面原滚动位置得到恢复。
2. 复制 Markdown 后粘贴到本地文本区，正文、图片和评论顺序与预览一致。
3. 非目标路由仍走既有通用提取，不误显示社区专用结构。
4. Popup 在 420x640 视口无横向溢出，长评论可正常滚动。
5. 控制台、content script 与 service worker 无新增 error。

## 探索性场景

- 图片帖无正文或评论为空时，仍可保存标题、来源和图片。
- 评论不足 50、恰好 50、超过 50 时，分别确认不补空项、不截断第 50 条、绝不输出第 51 条。
- 源站图片加载失败时正文和评论文本仍可复制/保存，不阻断整个剪藏。
- 打开小红书首页或贴吧列表页时，不误触发帖子专用提取器。
