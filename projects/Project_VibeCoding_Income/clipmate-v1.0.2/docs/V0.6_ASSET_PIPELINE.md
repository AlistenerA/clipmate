# V0.6 Asset Pipeline Foundation

## Scope

v0.6.0 建立图片保存的中间层，不改变现有 Notion 保存行为。

新增边界：
- `ClipAsset`：可保存资源的通用形态
- `FigureAsset`：文章正文图片、Markdown 图片、视频封面等图文资源
- `ImageSaveStrategy`：面向 Notion / Markdown 的保存策略评估
- `ImageAssetQualityReport`：图片保存质量报告，记录 ready / candidate / blocked 和原因

## Current Behavior

- 直接 http/https 图片 URL：Notion external image block ready
- data/blob/relative URL：blocked，并记录明确原因
- proxy/resize/api 型 URL：默认不作为 Notion external image；可标记为 File Upload external import candidate
- Markdown target：保留 external URL 引用，不受 Notion external 兼容性过滤影响

## Non-Goals

- 不下载图片
- 不上传图片
- 不缓存图片二进制
- 不新增 manifest 权限
- 不默认启用 Notion File Upload API
- 不创建 `clipmate-v0.6/` 目录的大规模复制提交

## Integration

`createNotionSavePlan()` 会从 draft body Markdown 中提取图片引用，并附带 `assetReport`：

```ts
{
  total: number
  ready: number
  candidates: number
  blocked: number
  issues: AssetQualityIssue[]
  strategies: ImageSaveStrategy[]
}
```

该报告当前只用于架构准备和测试验证，不改变 `buildNotionBlocks()` 的输出。
