# ClipMate v0.8 规划

更新日期：2026-06-19

## 实施状态

- v0.8.0 核心交付聚焦 Asset Picker；页面会话、覆盖层、草稿管理、Markdown/Notion 接入、自动化测试和人工测试文档已完成。
- v0.8.1 已修复人工测试发现的会话消费、Popup 关闭、模式切换与遮罩点击问题。
- v0.8.2 已修复 Runoob 代码容器、数字标题、Notion 列表与空图片题注问题。
- 详细补丁边界和 v0.9 页面感知模式方向见 `V0.8_X_PLAN.md`。
- Notion external image/video 的真实支持矩阵仍需使用测试凭据人工验证。
- File Upload external import、站点视频适配器和更细资源级重试未在本候选中启用，继续遵守“不默认下载/上传”的安全边界。
- Chrome/Edge/真实 Notion 验收通过前，状态为自动化候选而非最终冻结发布。

## 版本目标

v0.8 从“自动提取后修补”转向“自动提取 + 用户可控资源选择”。开发时另建 `codex/clipmate-v0.8-*` 分支和 `clipmate-v0.8/` 目录；v0.7 冻结后不在原目录继续叠加功能。

## P0：Asset Picker

1. Popup 发起一次性图片选择会话，content script 在当前页显示轻量覆盖层。
2. 鼠标悬停只高亮候选图片；点击加入草稿，`Escape` 或取消按钮必须清理所有页面 DOM 与监听器。
3. 候选 URL 复用现有 lazy/srcset/picture 解析和 Asset Pipeline，拒绝 `data:`、`blob:`、`javascript:`、追踪像素和隐藏元素。
4. Popup 提供图片预览、移除、排序和数量上限；选中结果只属于当前草稿，不写入长期站点缓存。
5. 继续使用现有 content script 匹配范围，不新增广域网络权限；消息协议增加 session id，防止旧 Popup 回包污染新草稿。

验收：BBC、Runoob、CCTV-like、Bilibili 各完成“点选、取消、重新打开 Popup、保存 Notion”回归；页面关闭或导航后不得残留覆盖层。

## P0：Notion 媒体可靠性

1. 用真实 Notion 测试页验证 external image/video 支持矩阵和错误码。
2. 评估 Notion File Upload API，仅在外链不可直接渲染且用户主动选择时启用；不默认代理、下载或上传全部页面资源。
3. 将图片失败与正文失败拆分，保留可重试的资源级报告。

## P1：视频与结构适配器

1. 将 v0.7.3 的通用视频页摘要升级为站点适配器，优先覆盖 Bilibili、YouTube、Vimeo、Youku、腾讯视频。
2. 对 Notion 支持的 provider 输出 native video；不支持的 provider 保留 bookmark，并明确显示原因。
3. 扩展代码、公式、表格与互动示例的结构恢复，不抓取字幕、登录态媒体或 DRM 内容。
4. v0.7.3 的“未知资源”测试诊断层在适配器稳定后删除或替换为正式的可选调试面板。

## P1：保存体验

1. 保存前显示 block 统计和媒体降级摘要。
2. 错误详情支持定位具体 block 类型，同时继续禁止记录 token、Page ID、响应正文和用户内容。
3. 改进 Notion 页面头部样式与作者、发布时间格式，但保持单一 metadata callout。

## 版本节奏

- `v0.8.0-alpha.1`：Asset Picker 会话与页面覆盖层。
- `v0.8.0-alpha.2`：草稿图片管理和 Asset Pipeline 接入。
- `v0.8.0-beta.1`：Notion 媒体策略与视频 provider 适配。
- `v0.8.0`：Chrome/Edge、真实 Notion、隐私与权限审查全部通过后冻结。
- `v0.8.1`：Asset Picker 人工测试阻断修复。
- `v0.8.2`：教程结构与题注保真修复；当前复测候选。
- `v0.8.3+`：仅在复测仍有 P0/P1 时创建，不扩展产品范围。

## 明确不做

- 不读取浏览器 cookie 或复用用户登录态调用第三方私有 API。
- 不下载视频、绕过 DRM、抓取付费内容或默认上传整页媒体。
- 不把第三方参考插件代码整段复制进主线。
