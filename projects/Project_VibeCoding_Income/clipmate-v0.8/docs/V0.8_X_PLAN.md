# ClipMate v0.8.x 补丁路线图

更新日期：2026-06-19

## 目标

v0.8.x 只处理 v0.8 人工测试暴露的可靠性与保真问题，不引入新的平台、权限、远程 API、存储 schema 或大产品模式。

## v0.8.1：Asset Picker 可靠性

状态：已实现并通过自动化。

- completed session 等待当前页草稿恢复后再消费，修复 Popup 重开仍为 0/20。
- 启动成功后自动关闭 Popup。
- 切换模式先取消 active picker，旧异步提取不能覆盖新模式。
- 卡片遮罩点击按候选矩形回落，覆盖 VCG-like 图集。
- 正文内大尺寸技术 logo 可手选，页头 logo、头像和图标继续过滤。

## v0.8.2：教程结构保真

状态：已实现并通过自动化，等待 Chrome/Edge/真实 Notion复测。

- Readability 保留 class，使 `example_code` 等窄规则能在完整提取链路中命中。
- 数字标题 `1\.` 规范为 `1.`。
- 教程列表进入 `ClipDocument` list block，并映射为 Notion 原生 bulleted/numbered list item。
- 无 alt、title、figcaption 的图片保持空 caption，不生成伪 `image` 题注。
- 新增 22 图上限、无候选和失效外链 fixture，降低人工测试门槛。

## v0.8.3 及后续补丁

### v0.8.3：全文结构与图片语义

状态：已实现并通过定向自动化。

- 全文表格、标题、列表、代码使用原生 Notion block。
- 真实题注去重，alt 不再冒充 caption，占位图片描述被清理。
- metadata callout 使用安全的网站图标 URL。

### v0.8.4-v0.8.5

#### v0.8.4：混合代码识别

状态：已实现并通过定向自动化与构建。

- 结构预处理保证短代码进入 code fence。
- 限定语言 highlight.js fast path，低置信度使用扩展本地 ML 模型。
- SyntaxHighlighter 布局表不再误识别为普通表格。

#### v0.8.5：交互与 History 收口

继续处理 Asset Picker 滚动悬停状态和 History 行为信息；完成后冻结 v0.8。

仅在 v0.8.2 复测仍发现发布阻断时创建，优先级如下：

1. P0：跨页草稿污染、覆盖层残留、正文丢失、非 HTTP(S) URL 进入保存链路。
2. P1：真实站点无法点选、代码/标题/列表仍明显失真、Notion 单图失败阻断正文。
3. P2：提示文案、局部间距、测试说明等不影响核心路径的问题。

不因为 P2 单独无限增加补丁版本；无 P0/P1 时冻结 v0.8.2。

## v0.9 产品方向：页面感知模式

该方向来自 Bilibili 人工测试，但不进入 v0.8.x：

- 根据已有 Page Type Detector 与站点 profile 推荐可用模式，而不是所有页面固定显示全文/选区/教程。
- 视频页优先显示“视频收藏”“教程”“神评/评论选区”，弱化或隐藏无意义的全文模式。
- 教程页优先教程模式；普通文章保留全文与选区；讨论页突出评论上下文。
- 自动推荐必须允许用户手动切换，并显示简短理由，避免黑盒误判锁死功能。
- 只使用当前 DOM、canonical/meta 与公开链接，不读取 cookie、登录态、字幕、DRM 媒体或第三方私有 API。

建议先做模式推荐与显隐，再单独设计视频收藏的数据模型和 Notion adapter。
