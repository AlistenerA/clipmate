## 1. 测试目标

验证 v0.8 Asset Picker 在 Chrome 与 Edge 中能够安全地从当前页面手动补充图片，并将选择结果正确合并到当前草稿、Markdown、Notion blocks 和 History。测试同时覆盖会话隔离、取消清理、Popup 重开、图片排序、数量上限、URL 安全过滤和 v0.7 功能回归。

## 3. 测试准备

1. 在 `clipmate-v0.8` 运行 `npm run build`。
2. 打开 `chrome://extensions` 或 `edge://extensions`，开启开发者模式。
3. 移除或停用旧 ClipMate，加载 `clipmate-v0.8/dist`，确认版本为 0.8.0。
4. 运行 `npm run qa:asset-picker`，打开 `http://127.0.0.1:4175/asset-picker.html`。
5. 另准备 BBC 文章、Runoob 教程、CCTV-like 图文页、Bilibili 视频页和一个测试 Notion 页面。
6. 测试记录不得包含 Notion token、Page ID、用户正文或完整 Markdown。

## 4. Asset Picker 核心用例

| ID | 操作 | 预期 |
|---|---|---|
| AP-01 | Popup 点击“选择页面图片” | 页面出现隔离工具条；显示候选数与 0/20 |

| AP-03 | 点击两张图，再次点击其中一张 | 首次加入并显示绿色序号；再次点击取消该图 |
| AP-04 | 点击“完成”，重新打开 Popup | 覆盖层全部清理；Popup 显示所选图片预览 |
| AP-05 | 按 Escape | 会话标为取消；覆盖层、工具条、事件监听全部清理；原草稿选择不变 |
| AP-06 | 工具条点击“取消” | 结果同 AP-05 |
| AP-07 | 选择中刷新、关闭或导航页面 | 新页面不残留覆盖层；旧结果不写入新草稿 |
| AP-08 | 完成旧会话后立即在另一页开始新会话 | 旧 session id 不得覆盖新页草稿 |
| AP-09 | 点击隐藏图、头像、追踪像素 | 不出现候选高亮，点击不加入草稿 |
| AP-10 | 点选懒加载图 | 使用 `data-src` 的真实 HTTP URL，不保存 data 占位图 |
| AP-11 | 连续选择超过 20 张 | 第 21 张不加入；计数不超过 20/20 |
| AP-12 | 受限页面或无候选页面启动 | 显示可理解错误，不留下覆盖层或半活跃状态 |

## 8. 回归与安全检查

- 全文、选区、教程提取、预览、复制、保存、History、重试均需通过。
- Options 的 token、目标 CRUD、默认目标、历史筛选与删除均需通过。
- Popup、Options、content script、service worker 控制台无新增错误。
- Manifest 只允许 `storage`、`activeTab`；host permission 只允许 Notion API。
- Asset Picker 不读取 cookie，不调用第三方私有 API，不下载/上传图片，不建立站点级长期缓存。
- 页面文字、token、Page ID、Notion 响应正文不得进入日志或测试记录。

- P1：四个真实站点矩阵无阻断问题。
- 任一覆盖层残留、跨页面草稿污染、非 HTTP(S) URL 进入保存链路或正文丢失均判定为发布阻断。
