# TEST_PLAN.md — ClipMate v0.2 测试计划

> 本轮为 v0.2 Session 6 发布前 QA，目标：确认 v0.2 所有功能可用，打包产物可正常加载。

---

## 测试环境

| 项目 | 要求 |
|------|------|
| 浏览器 | Microsoft Edge >= 126（Chromium 内核）|
| 扩展模式 | 开发者模式，加载解压缩的扩展 |
| 加载路径 | `clipmate-v0.2/dist/` |
| Notion Token | 有效的 Integration Token（[创建地址](https://www.notion.so/my-integrations)）|
| 目标 Page | Token 有写入权限的 Notion 页面 |
| 测试页面 | 见各用例说明 |

---

## 一、安装与加载

### TC-01：安装测试
**目标**：确认构建产物可被 Edge 加载。
1. 打开 `edge://extensions`，开启「开发人员模式」
2. 点击「加载解压缩的扩展」，选择 `clipmate-v0.2/dist/`
3. 预期：扩展出现在列表中，名称为「ClipMate」，版本 `0.2.0`，无红色错误提示

**通过条件**：扩展加载成功，无错误。

### TC-02：Edge 加载 dist 测试
**目标**：确认扩展所有入口页面可正常打开。
1. 在已加载的扩展中，确认工具栏出现 ClipMate 图标
2. 点击图标，Popup 正常弹出显示
3. 右键点击图标 →「扩展选项」，Options 页在标签页中打开

**通过条件**：Popup 和 Options 页面均可正常显示，无白屏、无崩溃。

---

## 二、Options 设置页

### TC-03：Notion Token 保存
**目标**：确认 Token 配置可正常保存和读取。
1. 打开 Options 页面
2. 填写有效的 Notion Token
3. 点击「保存」，预期显示「设置已保存」
4. 关闭并重新打开 Options，Token 应保持不变

**通过条件**：Token 保存后不丢失。

### TC-04：添加 Notion 目标页面
**目标**：确认多目标管理正常工作。
1. 在 Options 的「Notion 目标页面」区域点击「添加目标」
2. 填写目标名称和 Notion 页面 URL 或 Page ID
3. 点击保存
4. 预期：目标出现在列表中，首个目标自动设为默认

**通过条件**：目标添加成功，即时持久化。

### TC-05：编辑 Notion 目标页面
**目标**：确认目标可编辑。
1. 在目标列表中点击某个目标的编辑按钮
2. 修改名称或 Page ID
3. 点击保存
4. 预期：目标信息更新，关闭重开 Options 后保持不变

**通过条件**：目标编辑成功。

### TC-06：删除 Notion 目标页面
**目标**：确认目标可删除。
1. 在目标列表中删除一个非默认目标
2. 预期：目标从列表中移除
3. 删除默认目标
4. 预期：自动选择下一个目标为默认

**通过条件**：目标删除成功，默认目标逻辑正确。

### TC-07：设置默认目标
**目标**：确认可切换默认目标。
1. 在目标列表中点击"设为默认"
2. 预期：该目标标记为默认，之前默认目标的标记移除

**通过条件**：默认目标切换正常。

## 三、Popup 剪藏

### TC-08：Popup 默认选择默认目标
**目标**：确认 Popup 下拉框默认选中默认目标。
1. 在 Options 中设置一个默认目标
2. 打开任意网页，点击 ClipMate 图标
3. 预期：Popup 的目标下拉框默认选中默认目标

**通过条件**：默认目标正确选中。

### TC-09：Popup 切换目标保存
**目标**：确认可选择不同目标保存到不同 Notion 页面。
1. 在 Options 中添加至少两个目标
2. 打开 Popup，切换到第二个目标
3. 点击「保存到 Notion」
4. 预期：内容保存到第二个目标对应的 Notion 页面

**通过条件**：切换目标后保存到正确的 Notion 页面。

### TC-10：全文剪藏保存到 Notion
**目标**：确认全文模式正确提取并保存。
1. 打开一篇博客文章
2. 点击 ClipMate 图标，选择「全文」模式
3. 添加标签和备注，点击「保存到 Notion」
4. 预期：
   - 按钮显示「保存中...」→ 成功后显示「已保存到 Notion」
   - Notion 页面中标题、URL、标签、备注、正文结构完整

**通过条件**：全文剪藏保存成功，Notion 内容格式正确。

### TC-11：选区剪藏保存到 Notion
**目标**：确认选区模式正确提取并保存。
1. 打开任意网页，选中一段文字
2. 点击 ClipMate 图标，选择「选区」模式
3. 点击「保存到 Notion」
4. 预期：选区文字保存到 Notion 页面

**通过条件**：选区剪藏保存成功。

### TC-12：标签保存
**目标**：确认标签正确保存到 Notion。
1. 剪藏时添加标签（如 `技术,前端`）
2. 保存到 Notion
3. 预期：Notion 页面中标签以蓝色文字显示

**通过条件**：标签内容正确显示在 Notion。

### TC-13：备注保存
**目标**：确认备注正确保存到 Notion。
1. 剪藏时填写备注（如 `值得收藏`）
2. 保存到 Notion
3. 预期：Notion 页面中备注在 📝 callout 中

**通过条件**：备注内容在 callout 中正确显示。

### TC-14：复制 Markdown
**目标**：确认复制 Markdown 包含完整元数据。
1. 提取一篇文章，添加标签和备注
2. 点击「复制 Markdown」
3. 粘贴到文本编辑器
4. 预期：包含标题、URL、标签、备注、分割线、正文六部分，无异常粗体

**通过条件**：粘贴结果完整，无格式异常。

## 四、本地剪藏历史

### TC-15：保存成功写入历史
**目标**：确认保存成功自动写入历史记录。
1. 剪藏并成功保存到 Notion
2. 打开 Options →「剪藏历史」标签页
3. 预期：新记录出现在历史列表中，状态为「已保存」

**通过条件**：saved 状态历史记录正确生成。

### TC-16：保存失败写入 failed 历史
**目标**：确认保存失败写入历史并记录错误。
1. 使用无效 Token，尝试保存到 Notion
2. 打开「剪藏历史」标签页
3. 预期：出现 failed 状态记录，显示错误原因

**通过条件**：failed 历史正确记录。

### TC-17：历史搜索标题
**目标**：确认按标题关键词搜索。
1. 在历史搜索框输入文章标题关键词
2. 预期：匹配的历史条目显示，匹配文字高亮

**通过条件**：标题搜索正确，高亮渲染正确。

### TC-18：历史搜索 URL / 域名
**目标**：确认按 URL 域名搜索。
1. 搜索域名关键词
2. 预期：匹配的历史条目过滤显示

**通过条件**：URL/域名搜索正确。

### TC-19：历史搜索标签
**目标**：确认按标签搜索。
1. 搜索标签关键词
2. 预期：匹配条目显示，标签命中项高亮（黄底）

**通过条件**：标签搜索正确，命中高亮。

### TC-20：历史搜索正文，显示"正文匹配"
**目标**：确认正文搜索命中时显示提示 tag。
1. 搜索正文中的关键词（该词不在标题/URL/标签中）
2. 预期：条目显示橙色「正文匹配」tag

**通过条件**：正文匹配 tag 正确显示。

### TC-21：历史搜索备注，显示"备注匹配"
**目标**：确认备注搜索命中时显示提示 tag。
1. 搜索备注中的关键词
2. 预期：条目显示青色「备注匹配」tag

**通过条件**：备注匹配 tag 正确显示。

### TC-22：历史复制 Markdown
**目标**：确认可从历史重新复制 Markdown。
1. 在历史列表中找到一条已保存记录
2. 点击「复制 Markdown」按钮
3. 粘贴到编辑器
4. 预期：内容与原始剪藏时一致

**通过条件**：历史复制 Markdown 完整正确。

### TC-23：历史删除单条
**目标**：确认可删除单条历史记录。
1. 点击某条历史的删除按钮
2. 预期：该条目从列表中移除，storage 实时同步

**通过条件**：单条删除成功。

### TC-24：历史清空全部
**目标**：确认可一键清空所有历史。
1. 点击「清空全部历史」按钮
2. 确认弹出确认对话框
3. 确认后，预期：所有历史记录被清空

**通过条件**：清空全部成功。

### TC-25：failed / unsaved 历史重试保存
**目标**：确认可重试 failed 历史保存。
1. 找到一条 failed 历史记录
2. 选择目标后点击「重试保存」
3. 预期：
   - 保存成功后历史状态更新为「已保存」
   - 不会生成重复记录

**通过条件**：重试保存更新原记录。

## 五、History UI 体验

### TC-26：历史 favicon 优先显示真实图标
**目标**：确认历史条目显示页面真实 favicon。
1. 保存一篇有 favicon 的网页
2. 查看历史条目
3. 预期：条目左侧显示该网页的真实图标

**通过条件**：真实 favicon 正确渲染。

### TC-27：favicon 加载失败 fallback 到域名首字母头像
**目标**：确认 favicon 不可用时回退到首字母。
1. 保存一个没有 favicon 的网页（或 favicon 404）
2. 查看历史条目
3. 预期：条目左侧显示域名首字母圆形头像

**通过条件**：fallback 到首字母头像。

### TC-28：图片开篇文章历史摘要不显示 `![](...)` 
**目标**：确认摘要不显示 Markdown 图片语法。
1. 保存一篇开篇为图片的文章
2. 查看历史条目摘要
3. 预期：摘要不包含 `![]()` 或 `<img>` 标签，显示后续文字内容

**通过条件**：图片语法被清理。

### TC-29：同一网站多条历史左侧色条一致
**目标**：确认同站历史条目使用相同主题色。
1. 在同一网站保存多条内容
2. 查看历史列表
3. 预期：同一网站的所有条目左侧色条颜色相同

**通过条件**：同站色条一致。

### TC-30：同站不同文章打开 Popup 自动刷新
**目标**：确认切换文章后 Popup 自动提取新内容。
1. 在网站 A 的文章 1 处打开 Popup，关闭
2. 在同一网站 A 导航到文章 2
3. 重新打开 Popup
4. 预期：Popop 自动提取文章 2 的内容，不显示文章 1 的旧内容

**通过条件**：Popup 自动刷新。

### TC-31：同一页面重复打开 Popup 不丢失标签/备注
**目标**：确认草稿恢复不丢失编辑内容。
1. 在 Popup 中添加标签和备注，关闭 Popup（不保存）
2. 在同一页面重新打开 Popup
3. 预期：之前填写的标签和备注被恢复

**通过条件**：草稿恢复正确。

**目标**：确认未配置 Token 时给出正确提示。

2. 提取网页内容，点击「保存到 Notion」
3. 预期：显示「请先在设置中配置 Notion Token」

**通过条件**：显示中文错误提示。

### TC-35：目标缺失错误提示
**目标**：确认未配置目标时给出正确提示。
1. 删除所有 Notion 目标
2. 提取网页内容，点击「保存到 Notion」
3. 预期：显示「请先在设置中添加 Notion 目标页面」

**通过条件**：显示中文错误提示。

### TC-36：Notion API 失败错误提示
**目标**：确认 API 错误时给出正确提示。
1. 使用无效 Token 或无效 Page ID 尝试保存
2. 预期：显示具体错误提示，Token 不泄露

**通过条件**：友好错误提示，无敏感信息泄露。

## 八、隐私与打包

### TC-37：隐私检查
**目标**：确认控制台不泄露敏感信息。
1. 打开 DevTools（F12），切换到 Console 标签
2. 执行保存操作和错误场景测试
3. 检查控制台输出：
   - 不应出现明文 Token
   - 不应出现正文全文
   - 不应出现备注全文
   - 不应出现完整 Markdown
   - 可以出现错误码和字数统计

**通过条件**：控制台无敏感信息泄露。

### TC-38：Build 测试
**目标**：确认 `npm run build` 成功。
1. 运行 `npm run build`
2. 预期：tsc 无类型错误，vite build 成功
3. 检查 `dist/manifest.json` 存在且 version 为 `0.2.0`

**通过条件**：Build 成功，manifest version 正确。

### TC-39：Zip 测试
**目标**：确认 `npm run zip` 成功。
1. 运行 `npm run zip`
2. 预期：生成 `clipmate-v0.2.zip`

**通过条件**：zip 生成成功。

### TC-40：Zip 内容检查
**目标**：确认 zip 不包含不应提交的内容。
1. 检查 zip 内容
2. 应包含：manifest.json、service-worker-loader.js、assets/、src/（仅 popup/options 构建 HTML）、icons/
3. 应**不**包含：源码文件（.ts/.tsx 源文件）、tests/、docs/、node_modules/、.env、package.json、README.md、.git/

**通过条件**：zip 仅含 dist 构建产物。
