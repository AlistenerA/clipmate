# V0.3_HANDOFF.md — ClipMate v0.3 版本交接与规划草案

> 本文档不是 v0.3 具体开发 Prompt，也不是立即开发 v0.3。
> 它只是 v0.3 启动前使用的交接说明。
> v0.3 必须在用户明确要求后，从稳定 v0.2.x 目录复制创建，不直接在 clipmate-v0.2/ 中开发。

---

## 1. 当前状态

- v0.2 是轻量可用性增强版。
- v0.2 已实现多 Notion 目标、本地剪藏历史、保存状态、重试、历史搜索/复制/删除/清空等能力。
- v0.2 不应再继续膨胀为 AI 或多平台版本。
- v0.3 应从稳定的 v0.2.x 目录复制创建，不直接在 clipmate-v0.2/ 中开发。

---

## 2. v0.3 启动前必须完成

- v0.2 自动化测试通过。
- v0.2 人工 QA 完成。
- v0.2 RELEASE_CHECKLIST 完成。
- v0.2 是否发布到 Edge Add-ons 由用户决定。
- 如果 v0.2.1 有必要，应先做 v0.2.1 patch，再启动 v0.3。

---

## 3. v0.3 可能方向

以下只是候选方向，不代表全部都要做：

### 方向 A：AI 摘要与 AI 标签

可能功能：
- 用户自带 API Key。
- 对剪藏正文生成摘要。
- 根据正文推荐标签。
- 保存到 Notion 时可选择是否包含摘要。
- 历史记录中可保存 AI 摘要结果。

风险：
- 涉及外部模型 API。
- 涉及费用。
- 涉及隐私政策更新。
- 需要明确用户内容会被发送到模型服务。
- 需要防止 Token/API Key 泄露。

建议：
- 如果做 AI，优先做"用户自带 Key + 明确开关 + 默认关闭"。
- 不要在 v0.3 首轮就接多个模型供应商。
- 不要默认上传全文，必须由用户主动触发。

### 方向 B：Notion Database 属性映射

可能功能：
- 保存到 Notion database。
- 支持 title/url/tags/date/source/status 等属性映射。
- 用户在 Options 中配置 database id 和字段名。

风险：
- 配置复杂。
- Notion API 错误场景增多。
- 很容易让产品变重。

建议：
- 可作为 v0.3 或 v0.2.x 的独立小版本评估。
- 必须先做只读方案设计，不要直接实现。

### 方向 C：多平台保存

候选平台：
- 飞书
- 语雀
- Obsidian 本地 Markdown 导出
- Logseq / 本地文件

风险：
- 权限、鉴权、格式转换、隐私政策复杂度显著增加。
- 每个平台都需要单独测试和错误映射。

建议：
- v0.3 不要一次做多个平台。
- 优先评估 Obsidian / 本地 Markdown 导出，因为隐私和 API 风险更低。
- 飞书/语雀建议作为 v0.3+ 或 v0.4 再评估。

### 方向 D：OCR / 截图回退

可能功能：
- 对无法提取正文的网站使用截图或 OCR。
- 保存网页视觉内容。

风险：
- 权限更敏感。
- 性能和准确率不可控。
- 审核压力更高。
- 可能涉及图片隐私。

建议：
- 不建议作为 v0.3 首要功能。
- 除非用户明确把 OCR 作为核心方向，否则暂缓。

### 方向 E：付费 / License / 账号系统

可能功能：
- License Key。
- 订阅。
- 云同步。
- 团队功能。

风险：
- 需要后端、支付、用户账户、客服、隐私与合规。
- 明显超出当前轻量插件阶段。

建议：
- 不建议 v0.3 做。
- 最早 v0.4+ 再评估。

---

## 4. v0.3 推荐优先级

推荐优先顺序：

1. v0.3 Session 0：只做需求评估、版本目录创建方案、权限/隐私影响评估。
2. v0.3 Session 1：确定唯一主线方向，不允许 AI + 多平台 + OCR 同时开工。
3. v0.3 Session 2+：按选定主线拆小 Session。

当前建议：
- 首选方向：AI 摘要与 AI 标签，前提是用户明确接受外部 API 与隐私说明。
- 次选方向：Notion Database 属性映射。
- 保守方向：Obsidian / 本地 Markdown 导出。
- 暂缓方向：OCR、飞书/语雀多平台、付费系统。

---

## 5. v0.3 启动 Prompt 原则

v0.3 的第一个 Prompt 必须只做评估，不写代码。

必须先回答：

- 为什么要做 v0.3？
- v0.3 的唯一主线是什么？
- 是否新增权限？
- 是否新增外部 API？
- 是否传输新的用户数据？
- 是否需要更新隐私政策？
- 是否需要创建 clipmate-v0.3/？
- 从哪个稳定版本复制？
- v0.2 是否已经冻结？
- 是否需要先做 v0.2.1？

---

## 6. v0.3 禁止事项

- 不允许在未创建版本目录前直接修改 clipmate-v0.2/ 作为 v0.3。
- 不允许一次性实现 AI、多平台、OCR、付费。
- 不允许默认开启外部 API。
- 不允许保存或上传用户内容而没有明确说明。
- 不允许新增敏感权限而不更新权限说明和隐私政策。
- 不允许提交 Token、API Key、真实用户数据。
- 不允许 push。

---

## 7. v0.3 交接给 ChatGPT 审查的节点

建议每个关键节点都给 ChatGPT 审查：

1. v0.3 方向选择前。
2. v0.3 目录创建前。
3. v0.3 Session 0 完成后。
4. 每个功能 Session 完成后。
5. 修改权限或隐私政策前。
6. 发布前鲁棒性检查后。
