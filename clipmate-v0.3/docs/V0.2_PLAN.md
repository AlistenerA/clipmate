## 一、v0.2.0 产品定位

ClipMate v0.2.0 是 v0.1 的可用性增强版，目标是在**不新增权限、不新增外部服务、不引入 AI/多平台/付费/OAuth** 的前提下，增强日常剪藏效率。

核心价值：
- 用户可以查看最近剪藏过的内容
- 用户可以重新复制历史 Markdown
- 用户可以将不同内容保存到不同 Notion 页面
- 保存失败时可以保留记录并重试，减少内容丢失
- 保持本地优先，不新增后端和云同步

## 四、关键技术决策

| 编号 | 决策 | 原因 |
|------|------|------|
| D-036 | v0.2 不新增 manifest 权限 | 所有功能基于现有权限完成；新增权限增加审核风险 |
| D-037 | v0.2 继续使用 chrome.storage.local | 历史记录可能超过 sync 100KB 限制；无需网络同步 |
| D-038 | v0.2 继续只调用 Notion API，不新增第三方 API | 保持最小外部依赖 |
| D-039 | historyLimit 默认 100 | 平衡用户需求与 storage 压力；用户可在 Options 调整 |
| D-040 | 历史记录保存 markdown | 用于重新复制和重试保存；需在隐私政策中声明 |
| D-041 | 单条历史 markdown 建议限制约 50000 字符 | 防止单条记录过大；超出截断并标记 markdownTruncated = true |
| D-042 | 删除 Notion 目标时，不删除历史记录 | 历史记录保留 targetName 快照，保持数据完整 |
| D-043 | 历史记录保留 targetName 快照 | 目标被删除后仍可查看历史记录的来源目标名称 |
| D-044 | 重试保存时原目标不存在，要求用户重新选择目标 | 避免保存到错误目标，用户体验优先 |
| D-045 | package.json 和 manifest 版本号在 Session 6 才改为 0.2.0 | 避免开发过程中半成品混淆 |
| D-046 | History UI 放在 Options 页面内，不新增独立 history page | 避免 manifest 变化（不新增 options_ui / action 入口） |

## 五、数据结构草案

> 以下为草案，不实现代码。Session 1 中最终确定并实现。

```ts
type SaveStatus = 'unsaved' | 'saved' | 'failed';

interface NotionTarget {
  id: string;            // uuid，前端生成
  name: string;          // 用户自定义名称，如"工作笔记本"
  pageId: string;        // Notion 页面 32 位 ID
  isDefault: boolean;    // 是否为默认目标，Popop 默认选中
  createdAt: string;     // ISO 8601
  updatedAt: string;     // ISO 8601
  lastUsedAt?: string;   // ISO 8601，最后使用时间，用于 Popup 排序
}

interface ClipHistoryItem {
  id: string;                 // uuid，前端生成
  title: string;              // 网页标题
  url: string;                // 网页 URL
  mode: 'fullpage' | 'selection';
  tags: string[];             // 用户输入的标签
  notePreview: string;        // 备注前 100 字符（用于列表预览），完整备注不单独存储
  contentPreview: string;     // 正文前 100 字符（用于列表预览），完整正文在 markdown 字段
  markdown: string;           // 完整 Markdown（含标题/URL/标签/备注/分割线/正文），限 50000 字符
  markdownTruncated: boolean; // markdown 是否被截断
  wordCount: number;          // 字数（中英文合计）
  targetId?: string;          // 保存时选择的 NotionTarget.id
  targetName?: string;        // 保存时的目标名称快照
  saveStatus: SaveStatus;     // unsaved / saved / failed
  savedAt?: string;           // ISO 8601，成功保存时间
  errorCode?: string;         // 失败时的错误码（如 NOTION_TOKEN_MISSING、NOTION_API_ERROR）
  createdAt: string;          // ISO 8601，剪藏时间
  updatedAt: string;          // ISO 8601，最后更新时间
}

interface ClipMateSettingsV2 {
  version: 2;                   // 固定值，用于迁移检测
  notionToken?: string;         // Notion Integration Token（可选，始终存储用于 Notion API）
  notionPageId?: string;        // v0.1 兼容字段，暂时保留（迁移后不再主动使用）
  notionTargets: NotionTarget[]; // Notion 目标页面列表
  defaultTargetId?: string;     // 默认目标 ID，Popup 默认选中此项
  defaultTags: string[];        // 默认标签
  saveHistoryEnabled: boolean;  // 是否保存历史（默认 true）
  historyLimit: number;         // 历史记录上限（默认 100，最小 10，最大 500）
}
```

## 六、迁移策略

**v0.1 → v0.2 settings 自动迁移：**

1. **检测**：settings 中无 `version` 字段，或 `version < 2`，视为旧 settings。
2. **触发时机**：Options 页面加载或 Background Service Worker 首次读取 settings 时。
3. **迁移步骤**：
   - 读取当前 settings
   - 如果存在 `notionPageId`（旧单目标），创建一个 `NotionTarget`：
     - id: 生成 uuid
     - name: "默认 Notion 页面"
     - pageId: 旧 `notionPageId` 值
     - isDefault: true
     - createdAt / updatedAt: 当前时间
     - lastUsedAt: 当前时间
   - 如果不存在 `notionPageId`，创建空的 `notionTargets: []`
   - 保留旧 `notionPageId` 字段（兼容性，不删除）
   - 设置 `version: 2`
   - 设置 `saveHistoryEnabled: true`
   - 设置 `historyLimit: 100`
   - 写回 storage
4. **安全约束**：
   - 迁移前后不打印 Token。
   - 迁移失败不得破坏原始 settings。先深拷贝旧 settings，迁移后原子写回。如果写回失败，保留旧数据不变并 log 警告。
   - 迁移仅执行一次（version === 2 后跳过）。
5. **新用户**：首次安装时直接初始化 `version: 2` 默认结构，不经过迁移逻辑。

## 八、每个 Session 的交付标准

每个 Session 必须满足以下全部条件：

1. `npm run lint` — 0 errors, 0 warnings
2. `npm run test` — 全部通过
3. `npm run build` — 构建成功
4. 更新以下文档（按需）：
   - `docs/CURRENT_STATUS.md` — 当前进度和下一阶段
   - `docs/CHANGELOG_AGENT.md` — 修改文件清单和摘要
   - `docs/TEST_LOG.md` — 运行命令、结果、错误
   - `docs/ISSUES.md` — 未解决问题，不隐藏失败
   - `docs/DECISIONS.md` — 技术取舍记录
5. 不提交 `dist/`、zip、`node_modules/`、`.env`、工具目录
6. 不输出真实 Token、正文全文、备注全文
7. 发给 ChatGPT 审查后再决定是否 commit
